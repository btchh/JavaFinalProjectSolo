Hospital Management System Documentation

System Overview

The Hospital Management System (HMS) is a comprehensive Java-based application designed to manage various aspects of hospital operations. The system follows object-oriented principles and implements a service-oriented architecture to handle different hospital management tasks.

Architecture and Design

The system is organized into several key packages:

  • Entities/ - Contains domain model classes
  • Services/ - Houses business logic and service implementations
  • Interfaces/ - Defines contracts through interfaces
  • Enums/ - Contains enumeration types
  • Utils/ - Utility classes and helper functions
  • Main/ - Application entry point and control flow

The Four Pillars of Object-Oriented Programming

1. Encapsulation

Encapsulation in the HMS is implemented through private fields and public methods in entity classes. This principle is demonstrated in how the system handles sensitive data:

Example: Patient records are encapsulated within the Patient class, with private fields for personal information and public methods for controlled access and modification.

Benefits in HMS:

  • Data security for patient information
  • Controlled access to medical records
  • Validation of data modifications

2. Inheritance

The HMS utilizes inheritance to create a hierarchical structure of classes, promoting code reuse and establishing "is-a" relationships.

Example: Both Doctor and Nurse classes inherit from a common Person class, sharing basic attributes while adding their specific functionalities.

Implementation in HMS:

  • Common attributes shared through base classes
  • Specialized behavior in derived classes
  • Hierarchical organization of medical staff

3. Polymorphism

Polymorphism allows the HMS to handle different types of users and operations through common interfaces, making the system flexible and extensible.

Example: The system treats different types of medical staff polymorphically through a common interface, allowing for unified handling of appointments and records.

Applications in HMS:

  • Method overriding for specialized behavior
  • Interface implementation for different service types
  • Dynamic method dispatch in user operations

4. Abstraction

Abstraction is implemented through interfaces and abstract classes, hiding complex implementation details and providing clear, high-level interfaces.

Example: The AppointmentService provides a high-level interface for scheduling appointments without exposing the complex underlying logic of time management and conflict resolution.

Benefits realized:

  • Simplified interaction with complex systems
  • Reduced code complexity
  • Improved maintainability

Core Services

The HMS implements several key services:

  • AppointmentService: Manages scheduling and tracking of medical appointments
  • MedicalRecordService: Handles patient medical history and records
  • PrescriptionService: Manages medication prescriptions and tracking
  • AuthenticationService: Handles user authentication and security
  • RBACService: Implements role-based access control

Security and Access Control

The system implements robust security measures:

  • Role-based access control (RBAC) for different user types
  • Secure authentication mechanisms
  • Protected medical records access
  • Audit trails for sensitive operations
